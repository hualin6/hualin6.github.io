# ReactNative开发环境搭建——Windows/Android

# ReactNative

RN是 React的扩展包: 基于React 额外扩展了一套 手机端的组件库;

目前手机App混编的领域中, 占据很重要的地位:

主流的混编App方案

  • ReactNative: 基于React
  • Flutter: 基于Dart语言, 学习成本较高
  • UniApp: 基于vue的语法 和 微信小程序的组件库和api
  • ionic
  • 微信小程序
  • mintUI: 基于vue

ReactNative制作的程序可以编译成 能够安装到手机端上的 apk 然后安装在手机上!

ionic也支持: 但是要花钱

ReactNative必须手动搭建 编译环境: https://reactnative.cn/

搭建环境:

https://reactnative.cn/docs/getting-started.html

  • mac电脑 可以运行项目到 iOS手机上

以下讲解的是: windows 和 Android 手机搭配使用的编译环境搭建

mac电脑同学自己读官网: 自强! --- 超级容易, 只要网络好, 几个命令行就可以了!

也可以选择装windows系统

# Android Studio

编译到 Android 的apk 需要 编译工具: Android Studio

但是此工具比较大; 目前提取了此工具包中 编译用到的内容:

解压缩放到D盘根目录下: 路径要随着变化;

盘符可以变, 但是后期的设置需要 自行对应变更

添加 Android 编译工具的目录路径到 环境变量中:

填写 ANDROID_HOME

点击所有看到的确定按钮

重新打开环境变量: 发现已存在记录 就说明已保存

添加环境变量:

%ANDROID_HOME%\platform-tools

解释: 
%ANDROID_HOME% : 代表使用 刚才设置的 ANDROID_HOME 变量的路径

# 模拟器

rn开发的软件是 真正的 手机端App. 不是浏览器上执行的 混编App;

必须运行在手机端上;

  • 有Android手机同学: 可以直接真机执行
  • 没有Android手机同学: 可以使用模拟器执行

电脑上安装夜神模拟器 的版本比较低; 此处需要最新版本

先卸载 旧版本的夜神模拟器

安装新版本

  • 官网: https://www.yeshen.com/

  • 百度网盘上的也可以: 不是最新版本

安装方式: 一直下一步到最后即可!

启动后, 可以修改为手机版本

# 替换文件

夜神模拟器本身自带的 nox_adb 文件版本与 rn 不同, 会导致运行失败. 需要替换

具体步骤:

  • 关闭已开启的 夜神模拟器 窗口

  • 复制: D:\ANDROID_SDK\platform-tools\nox_adb.exe

    到底在哪个盘符 要看你之前解压的位置

  • 粘贴:

    • 在桌面上: 右键 夜神模拟器 图标: 选择 打开文件所在位置

    • 报错: 文件夹正在使用, 无法替换, 请重试

      有两种解决方案:

      • 通过 任务管理器 -> 在进程中 找到 nox 然后结束即可
      • 简单粗暴: 重启电脑

# 常见错误

  • nox_adb未响应
    • 关闭系统防火墙
  • ERROR: JAVA_HOME
    • java的环境变量不正确, Java安装不成功!
    • 到环境变量中查看 是否有 JAVA_HOME 字样, 有就删除掉.
    • 可能电脑之前配置过 java_home 导致的: 分析不出之前修改过什么; 但是重装系统肯定可以解决
  • Gradle 6.2 requires Java 8 or later to run. You are currently using Java 7
    • Java版本不正确, 按照文档的 java 部分进行
    • 通常是环境变量中, 有额外的设置: 设置了 java7 的目录导致的; 需要检查自己电脑的环境变量
  • 项目所在路径不允许中文

# 考虑到

不同的电脑本身存在:

  • 网络差: 编译时需要的资源下载慢 或者 无法下载 --- 无法编译成功!
  • 电脑本身环境乱: 你的电脑之前被其他人修改环境变量等内容; 常见是 java
    • 因为你也猜不到改了什么: 通常只能依赖重装系统来解决问题!!
