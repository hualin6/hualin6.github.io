# Vue-cli4

# 目录结构

//vue-cli4目录结构 //包括babel,webpack,vuex,vue-router
┌─dist	//npm run build打包生成目录//(webpack打包目录)
│        └─index.html	线上环境入口
├─node_modules	依赖包
├─public	用于存放静态文件
│        └─index.html	项目预览入口
├─src	//Vue文件名目录
│       ├─assets	静态文件目录
│       ├─pages 页面文件
│       │	└─index.js	
│       ├─App.vue	主vue模块 可引入其他模块
│       ├─main.js	Vue初始化入口文件(用于全局挂载实例和组件)
│       └─vue.config.js	//Vue配置文件//(用于代理等,要自己创建)
├─.gitignore	//git忽略上传目录
├─webpack.config.js	//webpack打包配置文件,要自己创建
├─babel.config.js	//babel代码转化es5配置文件
├─package.json	//包依赖组件(包版本,包内容)
├─package-lock.json	包依赖组件关联地址(限制包从哪个地址下)
└─ README.md	项目说明文档

# 3.html-webapck-plugin插件

作用

  • 为html文件中引入的外部资源如script、link动态添加每次compile后的hash，防止引用缓存的外部文件问题
  • 可以生成创建html入口文件，比如单页面可以生成一个html文件入口，配置N个html-webpack-plugin可以生成N个页面入口

简单来说就是它会自动地把我们要引入的文件打包进index.html文件，无需上面的通过 <script src="../dist/bundle.js"></script> 引入

使用方法

万事先安装

 //npm
 npm install html-webpack-plugin --save-dev
 //yarn
 yarn add html-webpack-plugin --save-dev
1

3
4

//webpack.config.js
module.exports = {
    ..................
    mode: 'development',
 <----------------------------------------------------------添加的部分开始----------------------------------------------------------->
//loader配置
    module: {
        //转换规则
        rules: [
            {
                //匹配后缀名为css的文件
                test: /\.css$/,
                use: [
                    //use数组里面loader的顺序是倒过来的
                    //将js的样式内容插入到style标签里(后加载)
                    "style-loader",
                    //将css文件转化为js(先加载)
                    "style-loader"
                ]
            },{
                //匹配图片文件
                test: /\.(jpg|png|gif)$/,
                loader: 'url-loader',
                //图片小于8kb,base64位处理(放入路径中),减少请求,但增加体积
                options: {
                    limit: 8 * 1024,
                    // url-loader的es6模块化解析关闭(防止与html的解析器冲突)
                    esModule: false,
                    //取图片哈希值的前十位
                    //[ext] 图片的扩展名
                    name: '[hash:10].[ext]'
                }
            },
            {
                //用于处理html
                test: /\.html$/,
                loader: 'html-loader'
            }
        ]
    },
<----------------------------------------------------------添加的部分结束----------------------------------------------------------->
	 plugins: [ 
        //实例化
        new HtmlWebpackPlugin({
            //以index.html文件作为模板去自动引入

# 本地模拟数据mock.js

好处

a.可以快速生成大量模拟数据

b.可以实现增删改查

坏处

数据得自己找(说是模拟,实际上已经是真数据)

# 1.在assets/js目录下新建Json.js(命名无要求,下面引入能对上就行)

├─assets 静态文件目录

│ ├─ js 脚本目录文件

│ │ └─Json.js Json数据.js文件

# 2.在 main.js 内引入

import Json from './assets/js/Json.js'

# 3.在 main.js 挂载到全局

//将这个请求封装为promise实例,使我们每次请求都可以重复复用
const json = type=>{
	//模拟异步请求数据(因为这个请求必定成功所以我们不用写reject的情况)
	return new Promise(resolve=>{
		//设定时器模拟真实环境延迟,一般在0.5~3秒内
		setTimeout(()=>{
			//如果promise请求成功,把Json内每项的  被导出  的数据赋给  json对象  同名属性
			//如被导出的是 A属性 ,实际上Json[type]就是 A属性
			resolve(Json[type]);
		}, 500)
	})
}
//挂载到vue全局以快速调用
Vue.prototype.$api = { json };
