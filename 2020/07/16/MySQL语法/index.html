单词/命令 用法/语义
-h: 服务器IP地址/域名
-P: 连接服务器的端口
-u: 用户名
-p: 用户密码
quit 退出服务器的连接
show databases 显示服务器上当前所有的数据库
use 数据库名称 进入指定的数据库
show tables 显示当前数据库中所有的数据表
desc 数据表名称 描述表中都有哪些列（表头）
set names UTF8 设置客户端连接服务端编码格式
drop database if exists 数据库名称 丢弃数据库 如果存在
create database 数据库名 charset=UTF8 创建新的数据库并设置存储编码格式
create table 表名称() 创建保存数据的表(包含表头及数据类型)
insert into 表 values() 向数据表中插入数据
update 表 set 值 where 条件 修改表中的值
delete from 表 where 条件 删除表中的行
PRIMARY KEY 主键约束(声明了主键约束的列上不允许插入NULL)
unique 唯一约束
not null 非空约束
default 默认值约束
check 检查约束
auto_incremnet 自增
foreign key(外键列) references 另一个表(主键列) 外键约束
select 字段1,字段2,... 选择查找字段
from 表名 来自某数据表
where 条件 查找条件
group by 分组条件 根据一个或多个列对结果集进行分组
having 筛选条件 根据条件筛选
order by 排序条件 顺序 将查询结果按指定条件升/降排序
limit 起始编号,数量 限制查询的记录数(每页显示几条记录)
max() 求最大值
min() 求最小值
avg() 求平均值
sum() 求和
count() 求总个数
between and 在...之间
in 在...之中
like 模糊查询(%,_) %:通配任意个字符, _ 通配任意一个字符
distinct 将查询结果去重
inner join 内连接
left outer join / left join 左外连接
right outer join / right join 右外连接
Union(左外和右外的结果组合) 合并相同的记录
Union all(左外和右外的结果组合) 不合并相同的记录
数 据 类 型
tinyint 微整型，占1个字节，范围-128~127 (数值型引号可加可不加)
smallint 小整型，占2个字节，范围-32768~32767
int 整型，占4个字节，范围-2147483648~2147483647
bigint 大整型，占8个字节
float 单精度浮点型，占4个字节，最大是3.4e38，可能产生计算误差
double 双精度浮点型，占8个字节，范围比bigint大的多，可能产生计算误差
decimal(M,D) 定点小数，几乎不会产生误差，M代表总的有效位数，D代表小数点后的有效位数
boolean/bool 布尔型，只有两个值，分别是true（真1）和false（假0），这两个值不能加引号
date 日期型 2020-01-25 (所有日期时间型必须加引号)
time 时间型 10:20:30
datetime 日期时间型 2020-01-25 10:20:30
varchar(M) 变长字符串，操作速度相对慢，M最大值是65535 (所有日期时间型必须加引号)
char(M) 定长字符串，操作速度相对快，M最大值是255，往往存储长度固定的数据，例如身份证号码，手机号码可以使用；可能产生空间浪费
text(M) 大型变长字符串，M最大值是2G

简      写:  mysql -uroot



查询示例



多条件查询: select id,name,salary from emp where id > 7 and salary > 10000; #条件是id大于7和工资大于10000 select name,title,salary from emp where salary between 10000 and 20000; #条件是工资10000-20000之间的 select id,name,salary from emp where salary in(7700,9000,11000); #条件是工资在7700,9000,11000的人员 模糊查询: select * from emp where name like '%a%'; #名字中带有a的人员 select * from emp where name like 'a%'; #名字是a开头的人员 select * from emp where name like '%a'; #名字是a结尾的人员 select * from emp where name like '%a_'; #名字中倒数第二个是a的人员 select * from emp where name like '_a%'; #名字中第二个是a的人员 分组查询: select title,avg(salary) from emp group by title; #平均工资,按职位排序 select mid,count(id),avg(salary),group_concat(name) as name from emp where mid = 3 or mid = 1 or mid = 8 group by mid having avg(salary) order by count(id) desc; #按职位排序,统计每个职位的人数和所有名字 select *from emp order by id; #将结果按工号升序排序 select *from emp order by id desc; #将结果按工号降序排序 select *from emp order by deptid desc,mid asc; #按多列排序:先按照deptid 降序排序,如果部门相同,再按照mid排序 limit 限制查询的记录数: select *from emp order by salary desc limit 0,5; #每五个数据一列,第一页 关键字的执行优先级: from → where → group by → having → select → distinct → order by → limit 解释: 1.找到表:from 2.拿着 where 指定的约束条件,去文件/表中取出一条条记录 3.将取出的一条条记录进行分组 group by ,如果没有 group by ,则整体作为一组 4.将分组的结果进行 having 过滤 5.执行 select 6.去重 7.将结果按条件排序: order by 8.限制结果的显示条数

创建数据库

SET NAMES UTF8; #设置客户端连接服务端编码 DROP DATABASE IF EXISTS sjk; #如果有sjk数据库，先丢弃 CREATE DATABASE sjk CHARSET=UTF8; #创建数据库sjk,并设置存储编码 USE sjk; #进入数据库 CREATE TABLE yg( id INT PRIMARY KEY, name VARCHAR(8), age INT ); #创建表yg,表里包含id,name,age,并设置数据类型 INSERT INTO yg VALUES(001,'ASD',30); #插入数据 INSERT INTO yg VALUES(002,'dfgd',20); INSERT INTO yg VALUES(003,'xfgdf',25); UPDATE yg SET age=18 where id=002; #将id为002的age改成18 DELETE FROM yg where id=003; #将id为003的数据删除
